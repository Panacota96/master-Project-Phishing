## Deliverable Checklist

### 1. A2 Poster
- [ ] Content finalized (see `01-poster/POSTER_CONTENT.md`)
- [ ] Screenshots captured for solution panels
- [ ] Architecture diagram exported from `documentation/ARCHITECTURE.md`
- [ ] QR code generated (live app URL or GitHub repo)
- [ ] PDF printed at A2 (420×594mm), high-resolution

### 2. Pitch Deck (10 min + 5 min Q&A)
- [ ] Slides built from `02-pitch/SLIDE_DECK_OUTLINE.md`
- [ ] Script rehearsed by both presenters (`02-pitch/PITCH_SCRIPT.md`)
- [ ] Q&A answers reviewed (`02-pitch/QA_PREP.md`)
- [ ] Slide deck exported to PDF as backup
- [ ] Timing tested (10 min max; leave 30 sec buffer)

### 3. Demo Stand (live or fallback)
- [ ] AWS environment confirmed live (`03-demo/DEMO_CHECKLIST.md`)
- [ ] Demo rehearsed end-to-end (`03-demo/DEMO_SCRIPT.md`)
- [ ] Fallback screenshots ready (`03-demo/FALLBACK_PLAN.md`)
- [ ] Screen recording exported as MP4 backup

---

## Day-of Logistics

- Arrive 30 minutes early to set up laptop + poster
- Room assignment posted on the communication office door
- Bring: printed A2 poster (rolled), laptop (charged + charger), HDMI adapter, phone as hotspot backup
- Pre-load both student and admin sessions in separate browser tabs before the jury enters
- If the room has no screen, use the poster as the primary visual
